# Kazi Rifat Al Muin - Online Judge Solutions

This repository contains my competitive programming solutions from online judges including Codeforces, AtCoder, CodeChef, CSES, Light OJ, UVa, and other practice sources.

## Website Dashboard

The repository includes a responsive HTML dashboard in `index.html` that shows:

- all indexed solution files
- folder-based statistics
- an all-folders summary strip at the top
- searchable and filterable file lists
- syntax-highlighted code viewing
- file size, language, modified date, and line count for the selected file

Run the dashboard from the repository root:

```powershell
npm run site:dev
```

This starts the local web server and watches the repository. When solution files are added, removed, renamed, or edited, `site-data.js` is regenerated automatically. The open website checks for fresh data every few seconds and updates its file list and stats.

Then open:

```text
http://localhost:8765
```

You can also run only the server manually:

```powershell
python -m http.server 8765
```

Then open:

```text
http://localhost:8765
```

## Keeping The Dashboard Updated

Regenerate the website data after adding, removing, or renaming solution files:

```powershell
npm run site:update
```

For automatic updates while working:

```powershell
npm run site:watch
```

The watcher refreshes `site-data.js` whenever repository files change. For the browser tab to update automatically, use `npm run site:dev` or keep a server running and leave the dashboard open.

## Housekeeping

This repository includes a `.gitignore` for common OS, editor, and build artifacts so the history stays focused on solution files and site scripts.

The dashboard data file, `site-data.js`, is generated by the helper scripts and kept in sync with the repository. After adding, removing, or renaming solution files, run:

```powershell
npm run site:update
```

If you want the dashboard to refresh automatically while you work, use:

```powershell
npm run site:watch
```
